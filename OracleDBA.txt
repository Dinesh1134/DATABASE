Assignment 1: Creating Tables with Primary Key & Foreign Key (Normalization)
Objective
To design a normalized database for a Student-Course Enrollment System and implement it using Oracle SQL with Primary and Foreign Keys.
Step 1: Identify Tables
Student
Course
Teacher
Department
Enrollment
Step 2: Apply Normalization
1NF
Separate data into different tables
No repeating groups
2NF
Each table has a Primary Key
3NF
Remove transitive dependency
Teacher linked to Department
Course linked to Teacher
Step 3: Create Tables
🔹 Department Table
SQL
CREATE TABLE Department (
    dept_id NUMBER PRIMARY KEY,
    dept_name VARCHAR2(50)
);
📸 Paste Screenshot: Department table created (DESC Department)
🔹 Teacher Table
SQL
CREATE TABLE Teacher (
    teacher_id NUMBER PRIMARY KEY,
    teacher_name VARCHAR2(50),
    dept_id NUMBER,
    FOREIGN KEY (dept_id) REFERENCES Department(dept_id)
);
📸 Paste Screenshot: Teacher table structure
🔹 Course Table
SQL
CREATE TABLE Course (
    course_id NUMBER PRIMARY KEY,
    course_name VARCHAR2(50),
    teacher_id NUMBER,
    FOREIGN KEY (teacher_id) REFERENCES Teacher(teacher_id)
);
📸 Paste Screenshot: Course table structure
🔹 Student Table
SQL
CREATE TABLE Student (
    student_id NUMBER PRIMARY KEY,
    student_name VARCHAR2(50)
);
📸 Paste Screenshot: Student table structure
🔹 Enrollment Table
SQL
CREATE TABLE Enrollment (
    student_id NUMBER,
    course_id NUMBER,
    PRIMARY KEY (student_id, course_id),
    FOREIGN KEY (student_id) REFERENCES Student(student_id),
    FOREIGN KEY (course_id) REFERENCES Course(course_id)
);
📸 Paste Screenshot: Enrollment table structure

SQL

Step 7: Verify Foreign Keys
SQL
SELECT constraint_name, table_name 
FROM user_constraints 
WHERE constraint_type = 'R';
📸 Paste Screenshot: Foreign keys list
Step 8: Final Output (Join Query)
SQL
SELECT s.student_name, c.course_name
FROM Student s
JOIN Enrollment e ON s.student_id = e.student_id
JOIN Course c ON e.course_id = c.course_id;
